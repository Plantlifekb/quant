📘 Quant v1.2 — User Guide
A governed, automated, end‑to‑end equity selection and backtesting platform

🧭 1. Overview
Quant v1.2 is a fully automated weekly equity‑selection and backtesting system.
It ingests market data, generates long‑only and long‑short selections, computes weekly returns, validates results against ingestion, and produces analytics for a multi‑page Streamlit dashboard.
The platform is designed to be:
- Modular — each stage is isolated and narratable
- Provenance‑preserving — every file has a clear source
- Audit‑ready — verification harnesses ensure correctness
- Automated — Task Scheduler runs the full pipeline weekly
- Dashboard‑driven — results are visualized in Streamlit

🏗 2. System Architecture
Quant v1.2 consists of six major components:
2.1 Ingestion Layer
- Script: scripts/ingestion/build_ingestion_5years.py
- Output: data/ingestion/ingestion_5years.csv
- Purpose: Pulls 5 years of daily OHLCV data for the trading universe.
2.2 Signal & Selection Layer
- Scripts:
- scripts/selection/build_weekly_selection.py (long‑only)
- scripts/selection/build_weekly_selection_longshort.py (long‑short)
- Outputs:
- data/signals/weekly_selection.csv
- data/signals/weekly_selection_longshort.csv
- Purpose: Generates weekly top‑ranked stocks and assigns weights.
2.3 Backtesting Layer
- Scripts:
- scripts/backtest/backtest_longonly_weekly.py
- scripts/backtest/backtest_longshort_weekly.py
- Outputs:
- data/analytics/quant_weekly_longonly_perf_v1.csv
- data/analytics/quant_weekly_longshort_perf_v1.csv
- Purpose: Computes weekly returns, cumulative returns, and drawdowns.
2.4 Verification Layer
- Scripts:
- scripts/verify_backtest_vs_ingestion_longonly.py
- scripts/verify_backtest_vs_ingestion_longshort.py
- Purpose: Recompute returns directly from ingestion and confirm exact match.
2.5 Combined Analytics Layer
- Script: scripts/analytics/build_combined_analytics.py
- Output: data/analytics/quant_combined_analytics_v1.csv
- Purpose: Produces unified long‑only, long‑short, and spread analytics.
2.6 Dashboard Layer
- Main app: scripts/dashboard/quant_dashboard_v1.py
- Pages:
- 1_Weekly_Picks.py
- 2_Performance.py
- 3_Regimes.py
- 4_Exposure.py
- 5_Diagnostics.py
- Purpose: Provides a multi‑page Streamlit interface for analysis.

- Verification

3.3 How to confirm the task is running
- Check Task Scheduler → History tab
- Check Last Run Time and Last Run Result (0x0)
- Check ingestion timestamp:
Get-Item C:\Quant\data\ingestion\ingestion_5years.csv | Select LastWriteTime

🧪 6. Verification & Auditability
Quant v1.2 includes two verification harnesses:
- Long‑only verification
- Long‑short verification
Each recomputes returns directly from ingestion and compares them to backtest output.
A full PASS means:
- No drift
- No mismatches
- No hidden transformations
- Fully narratable performance
This is the backbone of your governance.

🛡 8. Troubleshooting
Ingestion not updating
- Scheduler using wrong Python
- “Start in” not set to C:\Quant
- Script path incorrect
Dashboard not updating
- Combined analytics not rebuilt
- Cached data in Streamlit (use “Clear Cache”)
Verification fails
- Ingestion missing tickers
- Selection file corrupted
- Backtest logic modified
