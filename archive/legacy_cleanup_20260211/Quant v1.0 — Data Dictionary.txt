Quant v1.0 — Data Dictionary
Canonical, governed, non‑drifting

Purpose:
Define the meaning, units, constraints, and invariants for every
field in every governed CSV. This ensures semantic purity,
determinism, and auditability across the entire system.

──────────────────────────────────────────────────────────────
SECTION 1 — ENRICHED DAILY (enriched_daily.csv)

date
    Description: Trading date
    Format: YYYY‑MM‑DD
    Type: string
    Constraints: ISO‑8601, no nulls

ticker
    Description: Stock ticker symbol
    Type: string
    Constraints: uppercase, no spaces

company_name
    Description: Full company name
    Type: string
    Constraints: ASCII‑safe

market_sector
    Description: GICS or equivalent sector
    Type: string
    Constraints: lowercase, ASCII‑safe

open, high, low, close, adj_close
    Description: Daily OHLC prices
    Type: float
    Units: USD
    Constraints: non‑negative

volume
    Description: Daily trading volume
    Type: integer
    Units: shares
    Constraints: non‑negative

run_date
    Description: Date the enrichment was run
    Format: YYYY‑MM‑DD
    Type: string
    Constraints: ISO‑8601

ret_1d
    Description: 1‑day return
    Type: float
    Units: decimal (0.05 = 5%)
    Constraints: may be negative

ret_20d
    Description: 20‑day return
    Type: float
    Units: decimal
    Constraints: may be negative

vol_20d
    Description: 20‑day volatility
    Type: float
    Units: decimal
    Constraints: non‑negative

score
    Description: Quant v1.0 composite score
    Type: float
    Units: dimensionless
    Constraints: deterministic

mkt_avg
    Description: Market average return
    Type: float
    Units: decimal
    Constraints: deterministic

mkt_ma200
    Description: 200‑day moving average of market
    Type: float
    Units: USD
    Constraints: non‑negative

mkt_trend
    Description: Market trend classification
    Type: string
    Values: "bull", "bear", "neutral"

is_monday
    Description: Monday flag
    Type: integer
    Values: 0 or 1

SECTION 6 — GLOBAL INVARIANTS

- All dates must be ISO‑8601.
- All columns must be lowercase.
- No nulls permitted in governed fields.
- No blank columns permitted.
- No additional columns permitted.
- No reordering of columns permitted.
- All numeric fields must be parseable as float or int.
- All returns must be decimal format, not percent.
- All strings must be ASCII‑safe.
- All schemas must match the manifest exactly.
