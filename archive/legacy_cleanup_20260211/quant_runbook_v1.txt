Quant v1.0 — Operational Runbook
Canonical, governed, non‑drifting

1. Purpose
This runbook defines the operational procedures for executing, validating, and maintaining the Quant v1.0 governed pipeline. No deviations are permitted without a version bump.

2. Daily Responsibilities (Mon–Fri)
- Confirm ingestion_5years.csv is updated.
- Run enrichment_daily_quant_v1.py.
- Validate enriched_daily.csv via schema_validator_quant_v1.py.
- Review quant_v1.log for errors.

3. Weekly Responsibilities (Monday)
- Run ranking_weekly_quant_v1.py.
- Validate weekly_top10_quant_v1.csv.
- Publish dashboard_top10_quant_v1.csv.

4. Weekly Responsibilities (Saturday)
- Run backtest_weekly_quant_v1.py.
- Validate weekly_backtest_quant_v1.csv.
- Publish dashboard_growth_quant_v1.csv.

5. Full Pipeline Execution
Run the orchestrator:
    python C:\Quant\scripts\run_growth_pipeline_quant_v1.py

6. Validation Requirements
After every pipeline run:
- Run schema_validator_quant_v1.py.
- Run directory_auditor_quant_v1.py.
- Confirm no drift, no unexpected files, no legacy artefacts.

7. Logging Requirements
All modules must:
- Import get_logger from logging_quant_v1.py.
- Log start and end events.
- Log errors in governed format.
- Store logs in C:\Quant\logs\quant_v1.log.

8. Governance Rules
- No schema drift.
- No silent changes.
- No ungoverned files.
- No legacy artefacts.
- All dates ISO‑8601.
- All columns lowercase.
- Deterministic behaviour only.

9. Recovery Procedures
If any module fails:
- Stop the pipeline immediately.
- Inspect quant_v1.log.
- Re‑run the failing module manually.
- If failure persists, escalate and freeze outputs.

10. Versioning
Any change to:
- schema
- directory structure
- logic
- outputs
requires a version bump to Quant v1.1.

11. Audit Trail
All runs must produce:
- quant_v1.log entries
- validated CSVs
- no missing fields
- no blank columns
- no partial outputs

12. Operator Responsibilities
The operator must:
- Follow this runbook exactly.
- Never bypass validation.
- Never modify governed outputs manually.
- Maintain directory purity.
