Quant v1.0 — Turnover & Transaction Rules Specification

Canonical, governed, non‑drifting

Purpose

Define the exact turnover limits, transaction assumptions, rebalance mechanics, and invariants that govern all trading activity in Quant v1.0. This ensures that portfolio changes are deterministic, auditable, and free from discretionary or noise‑driven behaviour.

1. Transaction Philosophy

Quant v1.0 enforces:

deterministic weekly trading

zero discretionary overrides

zero randomness

zero adaptive turnover rules

zero transaction cost modelling

The system must behave identically for identical inputs.

2. Inputs

2.1 Required Files

weekly_top10_quant_v1.csv

weekly_backtest_quant_v1.csv

enriched_daily.csv

2.2 Required Fields

ticker

rank

score

ret_1w

cumulative_return

3. Rebalance Rules

3.1 Weekly Full Rebalance

Rebalance occurs:

every Monday

after ranking

3.2 Equal Weight Reset

All positions must be reset to: [ w_i = 0.1 ]

3.3 No Partial Rebalances

Forbidden:

threshold rebalancing

drift‑based rebalancing

intraday rebalancing

regime‑dependent rebalancing

4. Turnover Rules

4.1 Turnover Definition

Turnover is defined as: [ \text{turnover}(t) = \sum_{i=1}^{10} | w_i(t) - w_i(t-1) | ]

4.2 Expected Turnover

Because Quant v1.0 uses full weekly rebalancing:

turnover is typically 100%

turnover varies only when tickers remain in the top 10 across weeks

4.3 No Turnover Constraints

Forbidden:

turnover caps

turnover penalties

turnover‑based weighting adjustments

5. Transaction Assumptions

5.1 Zero Transaction Costs

Quant v1.0 assumes:

zero commissions

zero slippage

zero bid‑ask spread impact

5.2 No Shorting

All weights must satisfy: [ w_i \ge 0 ]

5.3 No Leverage

Total exposure must satisfy: [ \sum_{i=1}^{10} w_i = 1 ]

5.4 No Intraday Trading

All trades occur at the weekly rebalance.

6. Execution Rules

6.1 Deterministic Execution

Trades must be executed:

at the next available weekly open

using governed price data only

6.2 No Execution Modelling

Forbidden:

VWAP modelling

TWAP modelling

liquidity‑based execution

slippage modelling

7. Forbidden Adjustments

7.1 No Dynamic Position Sizing

Forbidden:

volatility targeting

risk parity

score‑weighted portfolios

regime‑dependent sizing

7.2 No Adaptive Turnover Rules

Forbidden:

turnover caps

turnover smoothing

turnover‑based filtering

7.3 No External Data

Forbidden:

fundamentals

analyst ratings

alternative datasets

8. Output

8.1 File

transactions_quant_v1.csv

8.2 Required Fields

ticker

weight_before

weight_after

turnover_contribution

9. Validation

9.1 Schema Validator

Ensures all transaction fields exist.

9.2 Compliance Checker

Ensures:

no leverage

no shorting

no forbidden turnover rules

no forbidden execution models

9.3 Directory Auditor

Ensures no alternative transaction files exist.

10. Failure Handling

If turnover or transaction rules are violated:

pipeline stops

no backtest may run

incident must be declared
