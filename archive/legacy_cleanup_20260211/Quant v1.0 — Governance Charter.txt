Quant v1.0 — Governance Charter
Canonical, governed, non‑drifting

Purpose:
Define the foundational principles, authority structure, and
non‑negotiable rules that govern the Quant v1.0 system. This
charter supersedes all other documents. No deviations permitted.

──────────────────────────────────────────────────────────────
1. Governance Philosophy

Quant v1.0 is built on four principles:
1. Determinism — same inputs produce same outputs.
2. Transparency — every transformation is narratable.
3. Purity — no ungoverned files, logic, or dependencies.
4. Integrity — no silent changes, no drift, no ambiguity.

These principles apply to:
- code
- data
- directory structure
- operators
- outputs
- validation
- logging
- incident response

──────────────────────────────────────────────────────────────
2. Authority Structure

The system is governed by:
- This Governance Charter
- The Master Manifest
- The Operational Runbook
- The Incident Response Playbook
- The Operator Certification Test

Authority hierarchy:
1. Governance Charter (highest)
2. Master Manifest
3. Runbook + Playbook
4. Validators
5. Modules
6. Operators (lowest)

Operators cannot override governance.

──────────────────────────────────────────────────────────────
3. Immutable Rules (Non‑Negotiable)

The following rules cannot be changed in Quant v1.0:

- No schema drift.
- No directory drift.
- No logic drift.
- No new dependencies.
- No manual edits to governed outputs.
- No unapproved files or folders.
- No silent failures.
- All dates ISO‑8601.
- All columns lowercase.
- All modules must log start/end.
- All outputs must pass schema validation.
- All runs must pass directory audit.
- All changes require version bump.

These rules are absolute.

──────────────────────────────────────────────────────────────
4. Versioning Requirements

Any change to:
- schemas
- directory structure
- logic
- dependencies
- governance rules
- module behaviour
- output format

requires:
- creation of Quant v1.1
- updated manifest
- updated runbook
- updated validators
- operator re‑certification

No exceptions.

──────────────────────────────────────────────────────────────
5. Determinism Requirements

Quant v1.0 must:
- produce identical outputs for identical inputs
- avoid randomness
- avoid non‑deterministic ordering
- avoid external data sources
- avoid time‑dependent behaviour except run_date

If determinism is violated:
- pipeline must stop
- incident must be declared
- escalation required

──────────────────────────────────────────────────────────────
6. Purity Requirements

The environment must contain:
- only governed directories
- only governed scripts
- only governed outputs
- only governed logs

Forbidden:
- temporary files
- legacy artefacts
- notebooks
- spreadsheets
- JSON, pickle, parquet
- any file not listed in the manifest

Purity is mandatory.

──────────────────────────────────────────────────────────────
7. Validation Requirements

Every run must pass:
- schema validation
- directory audit
- compliance check
- logging completeness check

If any validation fails:
- pipeline stops
- outputs freeze
- incident response begins

──────────────────────────────────────────────────────────────
8. Operator Responsibilities

Operators must:
- follow the runbook exactly
- never bypass validation
- never modify governed files
- never introduce new files
- never install new dependencies
- maintain directory purity
- escalate when required

Operators cannot:
- change logic
- change schemas
- change structure
- change governance

──────────────────────────────────────────────────────────────
9. Escalation Rules

Escalation is mandatory when:
- a module fails
- validation fails
- drift is detected
- forbidden files appear
- logs are incomplete
- determinism is violated

Escalation must include:
- failing module
- log excerpt
- timestamps
- operator name
- reproduction steps

──────────────────────────────────────────────────────────────
10. Charter Enforcement

This charter is enforced by:
- compliance_checker_quant_v1.py
- directory_auditor_quant_v1.py
- schema_validator_quant_v1.py
- operator certification
- incident response procedures

If any enforcement mechanism fails:
- system must freeze
- escalation required
- version bump required

──────────────────────────────────────────────────────────────
11. Charter Amendments

This charter cannot be modified in Quant v1.0.

Any amendment requires:
- Quant v1.1 creation
- updated manifest
- updated runbook
- updated validators
- operator re‑certification

──────────────────────────────────────────────────────────────
End of Governance Charter
