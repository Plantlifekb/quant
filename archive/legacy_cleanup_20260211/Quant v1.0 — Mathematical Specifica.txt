Quant v1.0 — Mathematical Specification

Canonical, governed, non‑drifting

Purpose

Define the exact mathematical formulas, invariants, and computational rules used across all Quant v1.0 modules. This specification ensures deterministic behaviour, auditability, and semantic purity.

1. Return Calculations

1.1 Daily Return (ret_1d)

[ ret_1d(t) = \frac{close(t) - close(t-1)}{close(t-1)} ] Constraints:

Decimal format

Deterministic

No smoothing

1.2 20‑Day Return (ret_20d)

[ ret_{20d}(t) = \frac{close(t) - close(t-20)}{close(t-20)} ] Constraints:

Requires full 20‑day history

No interpolation

1.3 Weekly Return (ret_1w)

[ ret_{1w}(t) = \frac{close(t) - close(t-5)}{close(t-5)} ] Assumes 5 trading days per week.

2. Volatility Calculations

2.1 20‑Day Volatility (vol_20d)

[ vol_{20d}(t) = \sqrt{\frac{1}{20} \sum_{i=1}^{20} (ret_1d(t-i) - \mu)^2} ] Where: [ \mu = \frac{1}{20} \sum_{i=1}^{20} ret_1d(t-i) ] Constraints:

Population standard deviation

No Bessel correction

3. Market Indicators

3.1 Market Average Return (mkt_avg)

[ mkt_avg(t) = \frac{1}{N} \sum_{i=1}^{N} ret_1d^{(i)}(t) ] Where:

(N) = number of tickers in universe

3.2 Market 200‑Day Moving Average (mkt_ma200)

[ mkt_{ma200}(t) = \frac{1}{200} \sum_{i=1}^{200} close_{mkt}(t-i) ] Constraints:

Deterministic

No exponential weighting

3.3 Market Trend Classification (mkt_trend)

Rules:

bull: (close(t) > mkt_{ma200}(t))

bear: (close(t) < mkt_{ma200}(t))

neutral: otherwise

4. Composite Score

4.1 Score Definition

Quant v1.0 score is defined as: [ score(t) = w_1 \cdot ret_{20d}(t) + w_2 \cdot z(vol_{20d}(t)) + w_3 \cdot z(ret_1d(t)) ] Where:

(w_1 = 0.5)

(w_2 = -0.3)

(w_3 = 0.2)

4.2 Z‑Score Normalization

[ z(x) = \frac{x - \mu_x}{\sigma_x} ] Where:

(\mu_x) and (\sigma_x) computed cross‑sectionally for the day

Constraints:

Deterministic

No winsorization

No clipping

5. Ranking Logic

5.1 Weekly Ranking

Tickers are ranked by descending score: [ rank(i) = 1 + \text{number of tickers with higher score} ] Constraints:

Stable sort

Deterministic tie‑breaking by ticker alphabetically

6. Backtest Calculations

6.1 Weekly Portfolio Return

[ ret_{1w}(t) = \frac{1}{10} \sum_{i=1}^{10} ret_{1w}^{(i)}(t) ] Where:

Top 10 tickers selected from weekly ranking

6.2 Cumulative Return

[ cum(t) = (1 + ret_{1w}(t)) \cdot cum(t-1) ] With: [ cum(0) = 1 ]

7. Dashboard Metrics

7.1 Weekly Return (dashboard)

Identical to backtest weekly return.

7.2 Cumulative Return (dashboard)

Identical to backtest cumulative return.

8. Global Mathematical Invariants

All returns must be decimal format.

All moving averages must be simple, not exponential.

All volatility calculations must use population variance.

All cross‑sectional statistics must be deterministic.

No randomness permitted.

No smoothing, clipping, or winsorization.

No external data sources.

End of Mathematical Specification
